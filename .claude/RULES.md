> Claude 행동 규칙 입니다.(Claude가 어떻게 작업할지, 참조 순서, 금지사항)

## 📚 참조 우선순위

작업 전 반드시 아래 파일을 순서대로 확인:

1. `@prompt/[해당-가이드].md` - 구체적인 코드 규칙
2. `@config/mvp-stack.yaml` - 기술 스택
3. `@config/design-system.md` - 디자인 시스템
4. https://v0.app/ - UI 디자인 레퍼런스

---

## 🗣 커뮤니케이션

- **언어**: 한국어 (코드 주석 포함)
- **응답**: 간결하게, 불필요한 설명 생략
- **질문**: 애매하면 먼저 확인 후 작업

---

## 🔄 작업 흐름

### 새 기능 개발 시

```
1. 요구사항 파악 → 2. 관련 규칙 확인 → 3. 구현 → 4. 테스트 안내
```

### 페이지 생성 시

```
"[페이지명] 페이지 만들어줘" 요청 시:
→ pages/[페이지명].html
→ css/[페이지명].css
→ js/[페이지명].js
3개 파일 일괄 생성 (mvp-rules.md 참조)
```

---

## ⚠️ 절대 금지

- ❌ `.env` 파일 수정
- ❌ `config/` 내 파일 임의 수정
- ❌ 기존 코드 삭제 (주석 처리로 대체)
- ❌ 확인 없이 대규모 리팩토링

---

## ✅ 필수 행동

- ✅ 파일 생성/수정 전 경로 확인
- ✅ 기존 코드 스타일 유지
- ✅ 변경사항 요약 제공
- ✅ 에러 발생 시 원인 + 해결책 함께 제시
